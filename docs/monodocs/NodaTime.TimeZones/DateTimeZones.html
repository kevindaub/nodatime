DateTimeZones Class

Static access to time zones by ID, UTC etc. TODO: Move these into DateTimeZone gradually.

public static class DateTimeZones

⊟ Remarks

The UTC time zone is defined to be present in all time zone databases and is built into the Noda Time system so it is always available. It is the only one guarenteed to be present in all systems.

All other time zones are acquired through providers. These are abstractions around the method of reading in or building the time zone definitions from the id. There is a list of providers and each is asked in turn for the matching time zone. If it does not have one the next is asked until one does return a time zone or the list is exhusted. Providers can use any mthod they want to resolve the time zone from the id.

The majority of time zones will be provided by the NodaTime.TimeZones.DateTimeZoneResourceProvider implementation which reads the time zones from an internal set of precompiled resources built using the NodaTime.ZoneInfoCompiler tool.

⊟ AddProvider Method

Adds the given time zone provider to the front of the provider list.

public static void AddProvider (IDateTimeZoneProvider provider)

⊟ Parameters

provider
The NodaTime.TimeZones.IDateTimeZoneProvider to add.

⊟ Remarks

Because this adds the new provider to the from of the list, it will be checked first for time zone definitions and therefore can override the default system definitions. This allows for adding new or replacing existing time zones without updating the system. If the provider is already on the list nothing changes.

⊟ Current Property

Gets or sets the current time zone.

public static NodaTime.DateTimeZone Current { set; get; }

⊟ Value

The current NodaTime.DateTimeZone. This will never be null.

⊟ Remarks

This is the time zone that is used whenever a time zone is not given to a method. It can be set to any valid time zone. Setting it to null causes the NodaTime.DateTimeZone.SystemDefault time zone to be used.
