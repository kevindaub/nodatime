Days Class

An immutable time period representing a number of months.

public sealed class Days : SingleFieldPeriodBase, IComparable<Days>, IEquatable<Days>

⊟ Remarks

Example 
Days
is an immutable period that can only store days.

It does not store years, months or hours for example. As such it is a type-safe way of representing a number of days in an application.

The number of days is set in the constructor, and may be queried using
Example
Value
. Basic mathematical operations are provided -

Example 
Days
is thread-safe and immutable.

ToStandardDuration () : NodaTime.Duration
This method allows you to convert between different types of period. However to achieve this it makes the assumption that all days are 24 hours long and all hours are 60 minutes long and all minutes are 60 seconds long. This is not true when daylight savings time is considered, and may also not be true for some unusual chronologies. However, it is included as it is a useful operation for many applications and business rules.

⊟ ToStandardDuration Method

This method allows you to convert between different types of period. However to achieve this it makes the assumption that all days are 24 hours long and all hours are 60 minutes long and all minutes are 60 seconds long. This is not true when daylight savings time is considered, and may also not be true for some unusual chronologies. However, it is included as it is a useful operation for many applications and business rules.
