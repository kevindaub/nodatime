Duration Struct

A length of time in ticks. (There are 10,000 ticks in a millisecond.)

public struct Duration : IComparable<Duration>, IEquatable<Duration>

⊟ Remarks

There is no concept of fields, such as days or seconds, as these fields can vary in length. A duration may be converted to an NodaTime.IPeriod to obtain field values. This conversion will typically cause a loss of precision.

This type is immutable and thread-safe.

⊟ ToPeriod Method

Converts this duration to a NodaTime.Periods.Period instance using the standard period type and the ISO calendar.

public NodaTime.Periods.Period ToPeriod ()

⊟ Returns

A Period created using the ticks duration from this instance

⊟ Remarks

Only precise fields in the period type will be used. Thus, only the hour, minute, second and millisecond fields on the period will be used. The year, month, week and day fields will not be populated.

If the duration is small, less than one day, then this method will perform as you might expect and split the fields evenly. If the duration is larger than one day then all the remaining duration will be stored in the largest available field, hours in this case. For example, a duration effectively equal to (365 + 60 + 5) days will be converted to ((365 + 60 + 5) * 24) hours by this constructor.

For more control over the conversion process, you must pair the duration with an instant
