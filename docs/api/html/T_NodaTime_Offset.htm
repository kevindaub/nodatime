An offset from UTC in milliseconds. A positive value means that the local time is ahead of UTC (e.g. for Europe); a negative value means that the local time is behind UTC (e.g. for America).

Remarks

Offsets are always strictly less than 24 hours (as either a positive or negative offset).

Internally, offsets are stored as an Int32 number of milliseconds instead of as ticks. This is because as a description of the offset of a time zone from UTC, there is no offset of less than one second. Using milliseconds gives more than enough resolution and allows us to save 4 bytes per Offset.
